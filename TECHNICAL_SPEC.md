# Технічне завдання: Vision Intelligence System Pro

## 1. Загальна інформація
**Vision Intelligence System** — це гібридний програмний комплекс для професійного моніторингу та аналізу відеопотоків у реальному часі. Система поєднує десктопну адмін-панель для локального керування (Qt) та адаптивний веб-сервіс для віддаленого доступу (Streamlit).

## 2. Ключові Функції
### 2.1. Візуалізація та Матриця
- **Dynamic Grid Engine:** Автоматичне розподілення до 4 потоків у рядок з підтримкою необмеженої кількості вікон.
- **Infinite Matrix:** Вертикальна та горизонтальна прокрутка для великої кількості камер.
- **Software PTZ:** Цифровий зум для кожного вікна окремо.
- **Auto-Scroll Trigger:** Автоматичне наведення на вікно, де відбулася важлива подія детекції.
- **Lazy Window Rendering:** Вікна камер створюються лише коли слот активовано.
- **Platform Streams:** Підтримка YouTube/платформ через yt-dlp (resolved URL cache).

### 2.2. Детектування та Аналітика (AI Engine)
- **Multi-Model Support:** Підтримка YOLOv8, YOLOv10, RT-DETR.
- **Cloud Models:** Можливість завантаження та застосування моделей через URL або файл.
- **Visibility Filter:** Налаштування видимості для кожного класу об'єктів окремо для кожної камери.
- **Night Boost:** Адаптивне покращення контрасту (CLAHE) для нічних умов.
- **Batch Folder Detection:** Обробка всіх файлів у папці з сортуванням за часом модифікації та експортом ZIP.
- **Per-slot Device Selection:** Вибір CPU/GPU для кожного слота з автофолбеком на CPU.

### 2.3. Керування Доступом та Користувачами
- **Role-Based Access (RBAC):**
    - **Admin:** Повний доступ, додавання джерел, зміна конфігурації.
    - **View:** Тільки перегляд доступних потоків.
- **User Control Table:** Адмін бачить список сесій і може миттєво змінювати роль, блокувати або видаляти користувача.
- **Auth System:** Доступ до веб-сервісу за паролем.
- **Real-time Web Controls:** Вибір джерел/моделей та запуск потоків напряму з Web UI.
- **Device Status:** Показ активного GPU/CPU у desktop і web інтерфейсах.

## 3. Технічні Характеристики
### 3.1. Системні вимоги
- **ОС:** Windows 10+, Linux (Ubuntu/Debian) з встановленими плагінами Qt.
- **GPU:** Рекомендується NVIDIA GPU з CUDA 11+ для високого FPS.
- **CPU:** Мінімум 4 ядра для 4 одночасних потоків.
- **RAM:** Мінімум 8 GB (рекомендується 16 GB).

### 3.2. Стек технологій
- **Мова:** Python 3.10+
- **GUI:** PySide6 (Qt for Python)
- **Web Node:** Streamlit
- **CV Engine:** OpenCV (Headless)
- **AI Backend:** Ultralytics (YOLO/RT-DETR)
- **Annotations:** Supervision
- **Storage:** JSON-based persistent state
- **Artifacts:** ZIP-архіви результатів batch-детекції (bbox overlays)

## 4. Схема Розгортання
1.  **Desktop Admin:** Основний процес, керує стрімінгом, аналітикою та веб-вузлом.
2.  **Web Node:** Дочірній процес Streamlit, забезпечує віддалений доступ через браузер (порт 8501 за замовчуванням).
3.  **State Sync:** Файл `system_state.json` синхронізує налаштування між десктопом та вебом у реальному часі.
